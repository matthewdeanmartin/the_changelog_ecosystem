# Claude Code — Project Notes

## Environment

- **Python is managed by `uv`.** Never use `pip`, `pip3`, `python`, or `python3` directly.
  All Python commands must go through `uv run …` or `uv sync`.
- **Task runner is `just`.** Run `just` to list all recipes. Common entry points:
  - `just build` — generate pages + build HTML
  - `just gather` — refresh tool metadata from registries
  - `just stubs` — generate stub review articles (never overwrites existing files)
  - `just devserver` — serve with live reload at http://localhost:8000

## Generated vs hand-written files

**Never hand-edit these** — they are fully regenerated by scripts and changes will be lost:

- `content/pages/tools.md`
- `content/pages/{ecosystem}.md` (python, rust, go, node, java, ruby, dotnet, cpp, cross)
- `data/tools.json` metadata fields (version, stars, last_release_date, archived)

**Safe to edit** — scripts will not overwrite these:

- `content/articles/*.md` (review stubs and finished reviews)
- `data/top_prio_tools.toml` (curated tool list)
- `content/pages/home.md`, `content/pages/about.md`
- `themes/` (templates and CSS)

## Key scripts

| Script | Purpose |
|--------|---------|
| `discover_tools.py` | Find new tools from registries (appends to tools.json) |
| `gather_metadata.py` | Fetch live version/stars/date/archived from APIs |
| `generate_pages.py` | Rewrite tools.md and ecosystem pages from tools.json |
| `generate_review_stubs.py` | Create stub articles for unreviewed tools |

All scripts support `--dry-run`. See `CONTRIBUTING.md` for full workflow.
